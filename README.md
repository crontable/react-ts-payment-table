# 주요 설계

- 주요 설계 의도와 트레이드오프 2~3개

1. 도메인을 먼저 파악하면서 데이터 렌더링
2. 시안대로 페이지로 구현
3. 컴포넌트 쪼개기
4. 검색 컴포넌트 + 검색 기능
5. 자잘한 스타일링 및 미비된 부분

# 요구 사항

## 테이블 값 채우기

- Ordered 영역
  - [x] consumption 항목을 그룹핑하여 Sub.TTL 계산된 수치를 출력한다.
  - [x] consumption 항목의 모든 총합(G.TTL)의 계산된 수치를 출력한다.
- Payable 영역
  - [ ] Payable 메타 정보(Due, Date, Attatchment, Memo) 항목에 알맞은 수치를 출력한다.
- Total 영역
  - [ ] Total 테이블에 알맞은 수치를 출력한다.

## 구조화

- [ ] 하나의 컴포넌트 안에서 데이터를 받아오면, 이를 렌더링하기 용이한 구조로 개편한다.
- [ ] 컴포넌트 내에서 데이터를 유연하게 사용할 수 있도록 구조화한다.
- [ ] 검색 필터링 기능을 추가했을 때의 상태를 구조화할 수 있는 상태로 개선한다. (검색 필터링 기능과 연계)

## 렌더링

- [ ] 각 표가 가로 스크롤이 가능하도록 개선한다.
- [ ] 검색 필터링 기능
